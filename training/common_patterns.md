# Common Coding Patterns

## 1) Hash Map
- Use when you need O(1) lookup by value/key.
- Typical problems: two sum, anagram, frequency counting.

## 2) Two Pointers
- Use left/right pointers to shrink search space.
- Typical problems: container with most water, sorted array operations.

## 3) Sliding Window
- Use for contiguous substring/subarray optimization.
- Maintain window validity, update answer each step.

## 4) Stack
- Use for matching/ordering constraints.
- Typical problems: valid parentheses, monotonic stack variants.

## 5) Binary Search
- Use when answer space is ordered.
- Keep one boundary style consistent to avoid bugs.

## 6) Tree Traversal
- DFS for recursion structure, BFS for level order.

## 7) Dynamic Programming
- Define state, transition, and initialization first.
